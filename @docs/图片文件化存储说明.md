# 图片文件化存储功能说明

## 📋 功能概述

已实现图片文件化存储功能，将图片从Base64内嵌方式改为独立文件存储，提升性能和可维护性。

## 🗂️ 存储结构

```
用户数据目录/
├── notebook.db          # SQLite数据库（存储图片路径引用）
├── config.json          # 配置文件
├── images/              # 图片文件目录
│   ├── img_1234567890_abc123.webp
│   ├── img_1234567891_def456.jpg
│   └── ...
└── notes/               # Markdown文件目录
    └── [分类名]/
        └── [笔记名]-[ID].md  # Markdown文件（包含图片路径引用）
```

## 🔧 核心功能

### 1. 图片上传与处理
- **文件格式转换**：支持转换为WebP、JPEG、PNG格式
- **图片压缩**：自动压缩大图片，减少存储空间
- **质量控制**：可配置压缩质量和最大尺寸
- **唯一命名**：使用时间戳+随机字符串生成唯一文件名

### 2. 图片信息管理
- **元数据存储**：保存图片ID、路径、尺寸、大小等信息
- **路径引用**：文档中存储图片路径而非Base64数据
- **文件验证**：检查图片文件是否存在

### 3. 自动清理机制
- **删除笔记时**：自动清理相关的图片文件
- **未使用检测**：识别并删除不再使用的图片文件
- **安全删除**：确保不会误删正在使用的图片

## ⚙️ 配置选项

```typescript
interface ImageStorageOptions {
  maxSizeMB?: number        // 最大文件大小（默认1MB）
  maxWidthOrHeight?: number // 最大宽高（默认800px）
  useWebWorker?: boolean    // 使用Web Worker（默认true）
  maxIteration?: number     // 最大压缩迭代次数（默认10）
  format?: 'webp' | 'jpeg' | 'png' | 'original' // 输出格式（默认webp）
  quality?: number          // 压缩质量（默认0.8）
}
```

## 🔄 工作流程

### 图片上传流程
1. 用户选择图片文件
2. 系统生成唯一图片ID和文件名
3. 压缩和格式转换处理
4. 保存到 `images/` 目录
5. 在文档中插入图片路径引用
6. 存储图片元数据信息

### 图片清理流程
1. 删除笔记时提取所有图片路径
2. 扫描 `images/` 目录中的所有文件
3. 对比正在使用的图片路径
4. 删除未使用的图片文件

## 🚀 使用方法

### 上传图片
```typescript
import { uploadAndSaveImage } from './services/imageStorage'

const imageInfo = await uploadAndSaveImage(file, {
  format: 'webp',
  quality: 0.8,
  maxWidthOrHeight: 800
})
```

### 清理未使用图片
```typescript
import { cleanupUnusedImages } from './services/imageStorage'

const usedPaths = ['path1', 'path2', 'path3']
await cleanupUnusedImages(usedPaths)
```

## 📝 注意事项

1. **Tauri环境**：图片文件化存储仅在Tauri环境中完全支持
2. **浏览器环境**：自动回退到Base64存储方式
3. **文件权限**：确保应用有读写图片目录的权限
4. **存储空间**：定期清理未使用的图片文件
5. **备份策略**：图片文件需要与数据库一起备份
