














**整体任务拆解（按阶段）**
1. **存储层迁移**
   - 梳理 `storage.ts` 内所有 SQLite 读写点，替换为“纯 Markdown 文件 + 文件系统 API”逻辑（含分类、标签、配置等数据的序列化格式）。
   - 清理 Tauri 中 `plugin-sql` 依赖及相关初始化（[main.rs](cci:7://file:///d:/AI/note_book/src-tauri/src/main.rs:0:0-0:0)、[Cargo.toml](cci:7://file:///d:/AI/note_book/src-tauri/Cargo.toml:0:0-0:0)、前端 [package.json](cci:7://file:///d:/AI/note_book/package.json:0:0-0:0)、类型定义等）。
   - 建立统一的文件结构约定：`root/categories.json`、`root/tags.json`、`root/notes/<category>/<slug-id>.md` 等，并实现 CRUD。

2. **内存加载/缓存机制**
   - 设计 `DocumentCache`：维护笔记元数据、内容、行偏移数组，支持懒加载与 LRU 驱逐。
   - 启动流程：扫描文件系统 -> 构建内存索引/缓存 -> 向 Zustand stores 注入初始数据。
   - 监听文件变更（`notify` crate 或 Tauri FS watcher）并同步到前端。

3. **Tantivy 索引集成**
   - `src-tauri` 引入 `tantivy` 依赖，定义 Schema（`title`,`content`,`path`,`line_offsets`）。
   - 实现索引构建器：遍历 Markdown -> 解析 Front Matter -> 写入索引；提供增量更新（新增/修改/删除）。
   - 暴露搜索命令（如 `search_docs(query) -> Vec<{ path, line, snippet }>`），支持分页与多字段筛选。

4. **前端/接口适配**
   - 更新 `storage`/`markdownStorage`/`noteStore` 调用链：改用新的文件存储与搜索命令。
   - 调整 UI：目录树直接读取文件系统；搜索面板展示 Tantivy 结果的文件+行号+片段。
   - 补充错误提示、加载状态及“手动重建索引”“刷新目录”等操作。

5. **测试与验证**
   - 单元/集成测试覆盖文件读写、缓存更新、搜索结果行号。
   - 手动验证：笔记 CRUD、跨平台路径、索引实时性（含大文件）。
   - 性能基准：首次索引时间、内存占用、查询延迟。

6. **收尾**
   - 更新文档（README/开发计划/配置说明），记录迁移步骤与回滚方式。
   - 清理遗留代码（SQLite 工具、旧搜索 API、无用类型）。
   - 梳理风险与后续优化项（如前端增量渲染、索引压缩）。
