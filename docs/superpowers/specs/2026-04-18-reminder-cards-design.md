# Reminder Cards Design

**Date:** 2026-04-18

**Status:** Approved for planning

## Goal

在现有 MuseNote 界面基础上新增一套简约的“提醒卡片”工作流，用来记录账号密码、服务器、域名或其他事项的到期时间与备注信息，并提供应用内筛选与桌面提醒能力，同时尽量不破坏现有文档编辑体验。

## Product Direction

本功能不再单独引入“提醒中心”底部入口，也不复用现有第二列文档列表作为提醒列表。最终交互如下：

- 第一列顶部新增 `文档 | 提醒` 模式切换
- `文档` 模式保持当前三栏布局不变
- `提醒` 模式下保留第一列分类，隐藏第二列，右侧主内容区变为提醒卡片墙
- 点击某张提醒卡片后，通过弹窗窗口进行编辑，而不是右侧详情页或卡片原地展开

## UX Scope

### Document Mode

`文档` 模式必须保持现状：

- 第一列为分类
- 第二列为文档列表
- 第三列为文档编辑器
- 现有标签页、富文本编辑、大纲、分类筛选逻辑不受影响

### Reminder Mode

`提醒` 模式的目标是让用户在同一屏看到多张提醒卡片并快速维护。

#### Layout

- 第一列：
  - 搜索框保留
  - `文档 | 提醒` 切换控件放在搜索框下方
  - 分类列表保留
  - 分类角标切换为“该分类下提醒卡片数量”
- 第二列：
  - 完全隐藏
- 右侧主区域：
  - 顶部工具条
  - 提醒卡片网格
  - 编辑弹窗由点击卡片触发

#### Top Toolbar

提醒模式右侧顶部工具条包含：

- 页面标题：`提醒卡片`
- 筛选标签：`全部`、`已过期`、`7天内`、`30天内`
- 右侧搜索框：用于搜索标题和备注
- `新建卡片` 按钮

不包含“当前分类统计”模块。

#### Card Grid

提醒卡片必须是简约样式，每行可显示多张，不固定为两列。布局需根据可用宽度自适应，例如：

- 常规宽度下 3-4 列
- 宽屏下更多列
- 保持统一卡片高度和整齐排列

每张卡片默认只显示：

- 标题
- 分类
- 到期日期
- 到期状态
- 备注预览（前两到三行）

不单独展示：

- 类型
- 地址
- IP
- 域名
- 账号
- 密码

这些内容统一写入备注字段。

#### Card Status

卡片状态由到期日期自动推导，主要对外展示以下几种：

- `已过期`
- `今天到期`
- `7天内`
- `30天内`
- `正常`

卡片主体配色保持克制，重点通过状态标签颜色区分优先级：

- 红色：已过期
- 橙色：7天内
- 黄色：30天内
- 灰蓝：正常

## Create and Edit Flow

### Create

新建提醒卡片流程必须尽量短：

1. 用户切到 `提醒` 模式
2. 点击右上角 `新建卡片`
3. 打开编辑弹窗
4. 填写最少字段
5. 保存后回到卡片墙

不引入模板、类型选择或多步向导。

### Edit Dialog

点击卡片后打开居中弹窗，背景卡片墙保持原位不跳动。弹窗字段只有：

- 标题
- 分类
- 到期日期
- 备注
- 启用提醒

弹窗底部操作：

- 取消
- 保存
- 删除

弹窗中需要提示用户可将额外信息写入备注，例如账号、密码、域名、IP、操作说明等。

## Reminder Rules

### Required Date

提醒卡片的到期日期为必填字段。因为此模式本身就是提醒工作流，若无到期日期则无法参与提醒与状态筛选。

### Reminder Toggle

卡片包含 `启用提醒` 开关：

- 开启时参与提醒与桌面通知
- 关闭时仍显示在提醒卡片中，但不触发通知

### Default Notification Points

默认提醒点固定为：

- 到期前 30 天
- 到期前 7 天
- 到期当天

第一期不支持用户自定义提醒点。

### Channels

提醒通过两条渠道触达：

- 应用内状态与筛选
- 桌面通知

每个提醒点只应触发一次。若用户修改了到期日期，则相关提醒状态需要重算。

## Data Model

第一期不引入独立的提醒数据表，直接在现有 `Note` 模型上扩展，以降低对现有架构的冲击。

### Existing Fields to Reuse

- `id`
- `title`
- `content`
- `categoryId`
- `createdAt`
- `updatedAt`

### New Semantics

- `title`：提醒卡片标题
- `content`：提醒卡片备注
- `categoryId`：所属分类

### New Fields

- `type: 'reminder-card' | existing note types`
- `dueDate?: Date`
- `reminderEnabled?: boolean`
- `reminderState?: { notified30d?: boolean; notified7d?: boolean; notifiedOnDay?: boolean }`

普通文档继续使用现有 `type`，提醒卡片统一使用 `type: 'reminder-card'`。

## State and Filtering

### UI Mode State

需要新增一个轻量 UI 状态，用于控制左侧切换：

- `document`
- `reminder`

此状态只影响界面布局和展示逻辑，不改变底层存储格式。

### Category Behavior

分类体系沿用现有 store。提醒模式下：

- 选中某分类时，右侧只显示该分类下的提醒卡片
- 若支持“全部分类”，则显示所有分类的提醒卡片

### Reminder Filters

右侧顶部筛选标签负责基于当前分类结果再做二次过滤：

- 全部
- 已过期
- 7天内
- 30天内

### Search

提醒模式下的搜索仅针对提醒卡片，并搜索：

- 标题
- 备注

## Component Boundaries

建议新增或修改的界面层职责如下：

- `Sidebar`：
  - 增加 `文档 | 提醒` 切换
  - 在不同模式下切换分类角标语义
- `App`：
  - 根据模式切换三栏布局与两栏布局
  - `文档` 模式继续渲染现有内容
  - `提醒` 模式渲染提醒卡片墙
- `ReminderBoard`：
  - 负责提醒模式的右侧主区域
  - 渲染顶部筛选、搜索、新建按钮和卡片网格
- `ReminderCardDialog`：
  - 负责新建和编辑提醒卡片

## Non-Goals for This Iteration

以下内容明确不在本次范围内：

- 字段级加密
- 类型化资产模型
- 服务器 / 域名 / 账号等专属字段表单
- 第二列提醒列表
- 原地展开编辑
- 底部“提醒中心”入口
- 当前分类统计面板
- 自定义提醒时间点

## Risks and Constraints

### Risk: Existing Layout Coupling

现有布局代码以三栏为主，提醒模式隐藏第二列时需要谨慎调整网格结构，避免影响文档模式。

### Risk: Mixed Note Types

提醒卡片复用 `Note` 模型会让同一 store 同时承载文档和提醒卡片，因此列表、搜索、删除等逻辑必须明确按 `type` 分流。

### Risk: Notification Repeat

如果提醒状态未单独记录，桌面通知可能重复弹出。因此提醒发送状态必须持久化。

## Validation Criteria

完工后应满足以下用户体验：

- 切换到 `文档` 模式时，应用表现与现状一致
- 切换到 `提醒` 模式时，第二列隐藏，右侧显示卡片墙
- 可在当前分类下快速新增提醒卡片
- 卡片列表一次可见多张
- 编辑通过弹窗完成
- 状态标签与筛选结果正确
- 到期前 30 天、7 天、当天可触发提醒
